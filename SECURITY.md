# Security Policy

## Reporting a Vulnerability

Please report security vulnerabilities privately.

Until a dedicated security contact is published, open a private security advisory
in GitHub for this repository. Do not post exploit details in public issues.

## Disclosure Process

1. Acknowledge report within 72 hours.
2. Reproduce and assess severity.
3. Prepare patch and tests.
4. Coordinate responsible disclosure and release notes.

## Scope

Security reports are especially valuable for:
- command execution isolation
- local socket authorization and abuse
- ACP permission bypass scenarios
- terminal escape and rendering attacks
